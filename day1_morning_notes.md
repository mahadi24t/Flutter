# 🚀 Flutter Journey - Day 1 (Morning Session)
**Topic:** Basic Widgets, UI Layouts, and Navigation

## 🧱 1. বেসিক উইজেট (UI Building Blocks)
ফ্লাটারে সবকিছুই একেকটা উইজেট (Widget), ঠিক যেমন ওয়েব ডেভেলপমেন্টে সবকিছু একেকটা HTML ট্যাগ। 

* **`Text('...')`**: স্ক্রিনে যেকোনো লেখা দেখানোর জন্য। (HTML-এর `<p>` বা `<h1>`-এর সমতুল্য)।
* **`Icon(Icons.name)`**: সুন্দর সুন্দর ভেক্টর আইকন বসানোর জন্য।
* **`SizedBox(height: ..., width: ...)`**: দুটো উইজেটের মাঝে অদৃশ্য ফাঁকা জায়গা তৈরি করতে। (CSS-এর `margin` বা `padding`-এর মতো কাজ করে)।
* **`CircleAvatar(...)`**: গোল প্রোফাইল ছবি বা আইকন বানানোর জন্য দারুণ একটি উইজেট।
  * `backgroundImage: NetworkImage('URL')` ব্যবহার করে সরাসরি ইন্টারনেট থেকে ছবি আনা যায়।
* **`ElevatedButton(...)`**: একটি ভাসমান বাটন বা বোতাম।
  * `onPressed: () { ... }` - এর ভেতরে বাটন ক্লিকের লজিক লিখতে হয় (JavaScript-এর `onClick` ইভেন্ট লিসেনারের মতো)।
* **`IconButton(...)`**: এটি একটি ক্লিকযোগ্য আইকন যা চাপলে সুন্দর রিপল/স্প্ল্যাশ (ripple/splash) ইফেক্ট দেয়। এটিও `ElevatedButton`-এর মতো `onPressed` প্রপার্টি ব্যবহার করে।

## 🏗️ 2. লেআউট উইজেট (Structuring the App)
একাধিক উইজেটকে স্ক্রিনে সুন্দরভাবে সাজানোর জন্য আমরা লেআউট উইজেট ব্যবহার করি। এগুলো অনেকটা CSS Flexbox-এর মতো কাজ করে।

* **`Column(children: [...])`**: উইজেটগুলোকে ওপর থেকে নিচে (Vertical) পরপর সাজাতে ব্যবহার করা হয়।
  * `mainAxisAlignment: MainAxisAlignment.center` দিলে কলামের ভেতরের সবকিছু লম্বালম্বিভাবে মাঝখানে চলে আসে।
* **`Row(children: [...])`**: উইজেটগুলোকে পাশাপাশি (Horizontal/side-by-side) সাজাতে ব্যবহার করা হয়।

## 📱 3. অ্যাপের মূল কাঠামো (App Architecture)
একটা পূর্ণাঙ্গ অ্যাপ দাঁড় করানোর জন্য কিছু বেসিক ফ্রেমওয়ার্ক বা মোড়ক লাগে।

* **`MaterialApp`**: পুরো অ্যাপের প্রধান মোড়ক। এটি ফ্লাটারকে বলে দেয় যে আমরা গুগলের ম্যাটেরিয়াল ডিজাইন (Material Design) ব্যবহার করব।
  * `debugShowCheckedModeBanner: false` - এটি ডানদিকের ওপরের বিরক্তিকর লাল 'DEBUG' ফিতাটি লুকিয়ে ফেলে।
* **`Scaffold`**: এটি হলো অ্যাপের মূল ক্যানভাস বা সাদা পাতা (HTML-এর `<body>`-এর মতো)। এর ভেতরেই আমরা অ্যাপের বাকি সবকিছু রাখি।
* **`AppBar`**: স্ক্রিনের একেবারে ওপরে যে হেডার বা নেভিগেশন বার থাকে।

## 🚪 4. নেভিগেশন (Routing between Screens)
ওয়েবসাইটে যেমন `<a>` ট্যাগ বা লিংক দিয়ে অন্য পেজে যাওয়া যায়, ফ্লাটারে সেই কাজ করে `Navigator`। এটি একটি স্ট্যাক (Stack) পদ্ধতিতে কাজ করে।

```dart
Navigator.push(
  context,
  MaterialPageRoute(builder: (context) => const ContactPage()),
);
```
push মানে হলো বর্তমান পেজের ওপর নতুন আরেকটা পেজ চাপিয়ে দেওয়া।

মজার বিষয় হলো, ফ্লাটার নিজে থেকেই তখন নতুন পেজে একটা 'Back Arrow' (পিছনে যাওয়ার বোতাম) তৈরি করে দেয়!

💡 5. প্রো-টিপস ও বেস্ট প্র্যাকটিস (Pro-Tips)
Custom Widgets: কোড পরিষ্কার রাখার জন্য আমরা StatelessWidget ব্যবহার করে নিজেদের পছন্দমতো কাস্টম লেগো ব্লক (যেমন- MyProfilePage, ContactPage) বানিয়ে নিতে পারি।

debugPrint vs print: অ্যাপের কনসোলে মেসেজ দেখার জন্য print ব্যবহার না করে debugPrint(...) ব্যবহার করাটা প্রফেশনাল ডেভেলপারদের নিয়ম।

Hot Restart: কোডে বড় কোনো পরিবর্তন করলে বা ফ্লাটার খেই হারিয়ে ফেললে (Error দিলে), অ্যান্টিগ্রাভিটি আইডিই-এর ওপরের সবুজ ঘোরানো তীর (Hot Restart) চেপে অ্যাপটি রিস্টার্ট করতে হয়।

Testing Files: একদম শুরুতে বেসিক শেখার সময় test ফোল্ডারের ভেতরের widget_test.dart ফাইলটি মুছে ফেলা ভালো, যাতে অন্য কোড ডিলিট করার কারণে অকারণে লাল এরর (Error) না আসে。
